Proposed flow of opeartions in API:
   1. User hits certain endpoint with a request to generate video. In this request user specifies
   whether they want this video to be published in one of their social networks or not and sets
   args for video generation
   2. API puts the task on a Queue and returns a UID of the task. Using this UID user's frontend 
   should be able to retrieve statuses from Redis Pub/Sub about their task
   3. Task is taken by one of the workers. Workers processes the task and pushes updates about the 
   process of video generation to Redis. Also, it pushes updates to the DB in some intervals.
   4. When the task of video geneartion is finished generated video is uploaded to MiniIO service. 
   MiniIO then generates a link that is saved in DB.
   5. Another task is created for publishing video on social networks. It's taken by another worker.

Goal for MVP:
One preset for video generation -> creating task over API -> generating video in background -> 
saving video to MiniIO -> writing the URL of generated video to DB -> pushing updates in real-time 
to Redis Pub/Sub -> publishing video to the choosen SN (one SN for MVP is enough)
